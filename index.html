System Design

Architecture

AI 파이프라인 아키텍처

Layered architecture: EvaluationController → NursingEvaluation OrchestrationService → Execution Layer (@Transactional) → AI Agent Components → Google Vertex AI (Gemini 2.0 Flash). All AI responses are validated via JSON Schema before being persisted to the database, ensuring 100% data integrity.
EvaluationController → NursingEvaluation OrchestrationService → 실행 레이어(@Transactional) → AI Agent → Google Vertex AI (Gemini 2.0 Flash) 순서로 처리됩니다. 모든 AI 응답은 JSON Schema 검증 후 DB에 저장되어 데이터 정합성을 보장합니다.
Core Features

EMR Simulation

실제 병원 EMR 환경 구현

Clinical Nursing EMR · 간호기록 (ENR) · GUIDED MODE
Students practice nursing assessments in a real hospital EMR environment. The system records Initial Assessment, Vital Signs (V/S), I/O, and Nursing Notes in real time. A step-by-step practice guide is shown on the right panel with live completion tracking.
실제 병원 EMR과 동일한 GUIDED 모드 환경에서 간호 초기사정, 활력징후(V/S), 섭취/배설(I/O), 간호노트를 실시간 기록합니다. 우측 패널에서 실습 진행률을 단계별로 확인할 수 있습니다.
검사결과 (Lab)
CBC, biochemistry, blood gas and coagulation results with trend charts. Critical values highlighted. Auto-generated nursing action reports.
CBC, 생화학, 혈액가스 결과를 추이 차트와 함께 제공. 위험 수치 강조 표시 및 간호 조치 보고서 자동 생성.
영상판독 (PACS)
X-Ray, CT, MRI, ultrasound images with diagnostic reports. Findings + Impression sections simulate real clinical reading.
X-Ray, CT, MRI 영상과 판독 보고서 제공. Findings/Impression 섹션으로 실제 임상 판독 시뮬레이션.
AI Pipeline

AI Evaluation

6개 AI Agent 실시간 자동 평가

AI 평가 실행 · Vertex AI (Gemini 2.0 Flash)
6 AI Agents (VitalSignsAgent, IoAgent, MedicationAgent, NursingRecordAgent, InitialAssessmentAgent, ActionLogAgent) evaluate 55 criteria simultaneously. The full AI prompt, patient context, and JSON response are logged transparently for professor review.
6개의 AI Agent가 55개 평가 항목을 동시에 평가합니다. AI 프롬프트, 환자 컨텍스트, JSON 응답이 모두 투명하게 기록되어 교수님이 검증할 수 있습니다.
교수용 평가 관리 대시보드 · Professor View
Professors review AI-generated scores with category-level feedback (correct / caution / needs improvement). Final scores can be approved or adjusted. Professor workload reduced by 40%, evaluation time by 90%.
교수님은 항목별 상세 피드백(잘한 점/개선할 점/주의사항)을 검토하고 최종 점수를 확정합니다. 교수 업무 부담 40%, 평가 시간 90% 단축 효과를 달성했습니다.
